# ADR-0006: Engineering OS v4.0 Architecture

## Status
Accepted

## Date
2026-07-12

## Context
Project Engine v2 contained gate-specific logic in switch statements.
Discovery, requirements, and brick planning required interactive terminal input.
No decision queue, dependency graph, or knowledge graph existed.

## Decision
Redesign as stateless orchestrator with dedicated gate handlers.
Every gate owns its workflow. AI-assisted discovery replaces interactive input.
Dependency graph and contract intelligence become mandatory before implementation.

## Architecture
Project Import → Governance → Gate Engine → Gate Handler → Discovery/Requirements/Architecture → Knowledge Graph → Dependency Engine → Brick Planner → Decision Queue → EM Approval → Contract Intelligence → Implementation → Validation → Release

## Principles
1. Project Engine is stateless — never contains gate logic
2. Every gate has a dedicated handler: .sdk/engines/gates/gate_N.sh
3. Discovery is AI-assisted, not interactive
4. Brick planning is automatic from dependency graph
5. Contract verification is mandatory before implementation
6. Decision queue surfaces only what needs human judgment
