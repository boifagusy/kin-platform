# ADR-0003: Session Lifecycle Amendment

## Status
Accepted

## Date
2026-07-11

## Context
Role assignment and project context were previously part of Gate 0 (Bootstrap).
This meant every new session required manual role assignment, and session state
was lost between Termux restarts. Governance decisions lacked persistent context.

## Decision
Move Role Assignment and Project Context out of Gate 0 into a new mandatory
Session Initialization Layer. Gate 0 becomes purely a project bootstrap gate.

## Architecture
ai session start
  ├── Project Detection
  ├── Session Recovery (restore previous state)
  ├── Role Assignment / Restore
  ├── Gate Recovery
  ├── Brick Recovery
  ├── Governance Load
  ├── Engine Health Check
  └── Session Ready → Governance → Execution

## Consequences
- Role persists across sessions — no repeated assignment
- Gate 0 is now pure Bootstrap (environment, SDK, governance, policy)
- Governance always has complete project context
- Session can auto-resume after Termux restart
- Breaking change: sessions must be initialized before any work
