# ADR-0007: Audit Platform & Investigation-First Workflow

## Status
Accepted

## Date
2026-07-12

## Context
The Engineering OS needed to move from a gate-based workflow to a fully
auditable engineering platform where every decision, investigation, and
certification is permanently recorded.

## Decision
1. Investigation Engine — No implementation begins before investigation
2. Certification Engine — Every task/brick/gate/release generates permanent artifacts
3. Mandatory workflow: Investigate → Verify → Plan → Approve → Implement → Validate → Certify → Commit → Release
4. Every Gate records entry/exit criteria, evidence, validation, certification, rollback, metrics
5. 10 permanent principles governing all engineering work

## Architecture
Project Engine (orchestrator only)
  ├── Governance Engine
  ├── Gate Engine
  ├── Investigation Engine ← NEW
  ├── Certification Engine   ← NEW
  ├── Contract Engine
  ├── Validation Engine
  ├── Audit Engine
  ├── Knowledge Engine
  └── Release Engine

## 10 Permanent Principles
1. Investigate before implementation
2. Verify before assumptions
3. One source of truth
4. Controllers coordinate only
5. Services own business logic
6. Presenters own formatting
7. Blade owns HTML
8. Every Gate is certified
9. Every Brick is independently releasable
10. Every decision is auditable
