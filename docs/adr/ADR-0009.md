# ADR-0009: Engine Manifest Specification

## Status
Accepted

## Date
2026-07-16

## Context
Engine responsibilities are currently inferred by grepping source code.
There is no standard way for an engine to declare its purpose, inputs,
outputs, commands, or dependencies. REGISTRY.yaml is becoming bloated.

## Decision
Every engine SHALL contain an `engine.yaml` manifest that declares:
- id, version, purpose, responsibility (single/shared)
- inputs, outputs, events
- commands exposed
- dependencies (depends_on, used_by)
- owner, status, lifecycle

REGISTRY.yaml SHALL contain only: engine ids, loading order, freeze status.
All other metadata lives in each engine's manifest.

## Consequences
- Engines become self-describing
- Documentation, dependency graphs, capability maps auto-generated
- No more grep-based responsibility detection
- REGISTRY.yaml stays small and focused
