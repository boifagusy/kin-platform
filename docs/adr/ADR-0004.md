# ADR-0004: Contract Registry & Implementation Lifecycle

## Status
Accepted

## Date
2026-07-12

## Context
Watchtower Dashboard implementation exposed governance gaps:
1. Architecture assumed service methods that didn't exist
2. No authoritative contract registry for service interfaces
3. Tasks had no validation/certification lifecycle
4. Architecture documents became stale after implementation

## Decision
Implement four new Engineering OS capabilities:
1. Service Contract Registry (.sdk/contracts/)
2. Contract Verification Engine
3. Task State Machine (Planning→Implementation→Validation→Certification→Checkpoint→Closed)
4. Architecture Synchronization Engine

## Principle
"No AI Assumptions" — Implementation must always be generated from verified contracts.
If contracts cannot be verified, BLOCK implementation.

## Consequences
- Every service interface must have a contract before implementation
- Contracts are verified against actual code before tasks proceed
- Architecture documents auto-sync with verified contracts
- Tasks cannot advance without certification
